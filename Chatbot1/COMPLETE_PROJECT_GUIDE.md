## 1. What Is This Project?

This is a **Hybrid Chatbot** — a web application that can answer questions in two ways:

| Mode | What It Does |
|------|-------------|
| **Direct Chat** | You type a question, and the chatbot answers it directly using a local AI model running on your computer. |
| **PDF Chat** | You upload one or more PDF files, the app reads them, indexes their content, and then answers your questions using **only** the information from those PDFs. |

The app runs entirely **on your local machine**. No internet connection is needed after setup. It uses **Ollama** to run AI models locally, **LangChain** to orchestrate the AI pipeline, **FAISS** to search through PDF content, and **Streamlit** to provide the web interface.

## 3. Technologies & Libraries Used

### Core Technologies

| Technology | Why We Use It |

## 4. Prerequisites & Installation

### Step 1: Install Python 3.12 or newer

Download from [python.org](https://python.org) and install. Verify:

```powershell
python --version
```

### Step 2: Install uv (recommended package manager)

```powershell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
```

Verify:

```powershell
uv --version
```

### Step 3: Install Ollama

Download from [ollama.com](https://ollama.com) and install.

### Step 4: Pull the required AI models

Open a terminal and run:

```powershell
ollama pull llama3.2
ollama pull nomic-embed-text
```

Verify both are installed:

```powershell
ollama list
```

You should see `llama3.2` and `nomic-embed-text` in the list.

### Step 5: Start Ollama

Make sure Ollama is **running** in the background before starting the app. On Windows, Ollama usually runs as a system tray app. You can also start it from the Start Menu.

### Step 6: Set up the project dependencies

Navigate to the project folder and run:

```powershell
uv sync
```

This reads `pyproject.toml` and installs all required packages into a local `.venv` folder.

### `get_llm()` (Lines 36–43)

```python
def get_llm():
    return ChatOllama(model=LLM_MODEL, temperature=0.3)
```

**What it does**: Creates the local chat model (`llama3.2`) with low temperature (0.3) for focused, less random answers.

**Why we need it**: Every time the app wants to ask the AI a question, it needs a model object. This function creates it consistently.

---

### `get_embeddings()` (Lines 47–52)

```python
def get_embeddings():
    return OllamaEmbeddings(model=EMBED_MODEL)
```

**What it does**: Creates the embedding model (`nomic-embed-text`) that converts text into numeric vectors.

**Why we need it**: To search through PDF chunks, we first convert them into vectors (numbers that represent meaning). This function gives us the tool to do that.

---

### `get_pdf_text(pdf_docs)` (Lines 56–76)

```python
def get_pdf_text(pdf_docs):
    text = ""
    for pdf in pdf_docs:
        pdf_reader = PdfReader(pdf)
        for page in pdf_reader.pages:
            page_text = page.extract_text()
            if page_text:
                text += page_text
    return text
```

**What it does**: Takes uploaded PDF files, opens each one, reads every page, extracts the text, and combines it into one big string.

**Why we need it**: PDFs are binary files. We need to extract the human-readable text before we can process it.

---

### `get_text_chunks(text)` (Lines 80–93)

```python
def get_text_chunks(text):
    text_splitter = RecursiveCharacterTextSplitter(
        chunk_size=1000,
        chunk_overlap=200,
    )
    chunks = text_splitter.split_text(text)
    return chunks
```

**What it does**: Takes the big block of PDF text and breaks it into smaller pieces (chunks). Each chunk is ~1000 characters, and consecutive chunks overlap by 200 characters.

**Why we need it**: AI models can't process entire PDFs at once. Smaller chunks are easier to search and stay within the model's context window. The overlap ensures no information is lost at chunk boundaries.

---

### `get_vector_store(text_chunks)` (Lines 97–110)

```python
def get_vector_store(text_chunks):
    embeddings = get_embeddings()
    vector_store = FAISS.from_texts(text_chunks, embedding=embeddings)
    vector_store.save_local(FAISS_PATH)
```

**What it does**: Takes the text chunks, converts each one into a vector (embedding) using `nomic-embed-text`, stores them in a FAISS index, and saves that index to the `faiss_index` folder on disk.

**Why we need it**: Once we have vectors, we can search them semantically — finding chunks that are *about* the same topic as the user's question, even if they don't use the exact same words.

---

### `get_conversational_chain()` (Lines 114–167)

```python
def get_conversational_chain():
    prompt_template = """
Answer the question as detailed as possible using only the provided context.

If the answer is not available in the context, respond with:

"Answer is not available in the context."

Do not make up information.

Context:
{context}

Question:
{question}

Answer:
"""
    model = get_llm()
    prompt = PromptTemplate(template=prompt_template, input_variables=["context", "question"])
    
    chain = (
        RunnablePassthrough.assign(
            context=lambda x: "\n\n".join(doc.page_content for doc in x["input_documents"])
        )
        | prompt
        | model
        | StrOutputParser()
    )
    return chain
```

**What it does**: Builds a processing pipeline (chain) that:
1. Takes the retrieved PDF chunks and joins them into a single context string
2. Fills the prompt template with that context and the user's question
3. Sends the formatted prompt to `llama3.2`
4. Converts the model's response into plain text

**Why we need it**: This is the heart of the RAG system. Without this, the model wouldn't know which PDF content to use when answering.

---

### `ask_pdf_question(user_question)` (Lines 171–228)

**What it does**: 
1. Checks if the FAISS index exists
2. Loads the FAISS index from disk
3. Searches for the 4 most similar chunks to the user's question
4. Builds the conversational chain
5. Runs the chain with the retrieved chunks and question
6. Displays the answer in the Streamlit UI

**Why we need it**: This is the function that actually performs PDF-based Q&A. It connects all the pieces: index loading, similarity search, and answer generation.

```
         │

         │
         ▼

```

## 8. Understanding RAG (Retrieval-Augmented Generation)

RAG is the technique used in PDF Chat mode. Here's why we need it:

### The Problem
If you ask a direct question to an AI model about a specific document, the model may:
- **Not know** the document's content (it wasn't trained on your PDFs)
- **Make up** information (hallucination)
- **Give generic** answers instead of document-specific answers

### The Solution (RAG)

RAG solves this by:
1. **Indexing**: Pre-process PDFs by splitting them into chunks and converting each chunk into a vector (embedding)
2. **Retrieval**: When a question is asked, find the chunks whose vectors are most similar to the question's vector
3. **Generation**: Send those retrieved chunks + the question to the LLM, and instruct it to answer using only that context

### Why Chunking & Embeddings?

| Step | Why |
|------|-----|
| **Chunking** | LLMs have a limited context window. They can't process an entire PDF at once. Small chunks (~1000 chars) fit easily. |
| **Overlap (200 chars)** | Ensures ideas that span across chunk boundaries are not lost. |
| **Embeddings** | Convert text into numerical vectors that capture *meaning*. This allows semantic search — finding chunks that are *about* the same topic, not just containing the same keywords. |
| **FAISS** | A highly optimized library for searching through millions of vectors quickly. Without it, searching through embeddings would be slow. |

## 9. Troubleshooting Common Issues

### Issue: "Streamlit command not found"

**Solution**: Use uv to run it:

```powershell
uv run python -m streamlit run app.py
```

Or make sure dependencies are installed:

```powershell
uv sync
```

---

### Issue: "Ollama model not found"

**Solution**: Pull the required models:

```powershell
ollama pull llama3.2
ollama pull nomic-embed-text
```

Verify they're installed:

```powershell
ollama list
```

Also ensure Ollama is **running** (check system tray or restart Ollama).

---

### Issue: "Please upload PDF files and click 'Submit & Process' first"

**Solution**: You asked a PDF Chat question before processing any PDFs. Upload at least one PDF and click the "Submit & Process" button in the sidebar, then ask your question.

---

### Issue: "No readable text found in the uploaded PDFs"

**Solution**: Your PDF is likely a scanned document (images of pages) rather than a text-based PDF. PyPDF2 cannot perform OCR. Use a text-based PDF instead, or convert your scanned PDF using OCR software first.

---

### Issue: run.bat does not work

**Solution**: The batch file points to a folder named `ragenv` which doesn't exist. Either:

1. Use the uv command:
   ```powershell
   uv run python -m streamlit run app.py
   ```

2. Or update `run.bat` to use `.venv`:
   ```bat
   @echo off
   echo Starting Streamlit app...
   call "%~dp0.venv\Scripts\activate.bat"
   "%~dp0.venv\Scripts\python.exe" -m streamlit run "%~dp0app.py"
   pause
   ```

---

### Issue: PDF Chat gives wrong or irrelevant answers

**Possible causes**:
- The uploaded PDFs don't contain the answer
- The chunk size (1000 chars) may be too small or too large for your specific documents
- The embedding model `nomic-embed-text` may not capture domain-specific terminology well

**Solutions**:
- Try uploading more specific/complete PDFs
- Adjust `chunk_size` and `chunk_overlap` in `get_text_chunks()`
- Try a different embedding model (update `EMBED_MODEL` and rebuild the index)

---

### Issue: The app is slow

**Possible causes**:
- `llama3.2` is a 3B parameter model — it runs reasonably fast on most modern CPUs/GPUs
- FAISS search on small indexes should be instant
- First-time loading may take longer

**Solutions**:
- Ensure no other heavy applications are running
- Consider using a smaller model like `llama3.2:1b` if speed is critical
- Make sure Ollama is using GPU acceleration if available
